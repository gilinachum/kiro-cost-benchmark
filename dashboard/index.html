Methodology

Same Model

Both tools use the same underlying Claude model. Differences measure harness efficiency, not model intelligence.

Fresh Session

Verbatim prompt, clean directory, no context carryover. No MCP tools, no custom instructions.

Automated Scoring

Binary pass/fail via pytest. 71 unit tests covering 7 modules. No subjective grading.

Cost Measurement

Claude Code: Bedrock API cost from output JSON. Kiro: credit count × $0.02 (Pro subscription rate).

Reproducible

All scenarios, tests, and runner scripts are open source. Pin versions, run yourself.

Limitations

N=1 per model (need N=5). Scenario is easy (100% pass). Harder scenarios planned.

Kiro Credit Multipliers

  • Haiku 4.5: 0.40× credits → cheapest option
  • Sonnet 4 / 4.5 / 4.6: 1.30× credits
  • Opus 4.5–4.8: 2.20× credits → narrows the gap
  • DeepSeek 3.2: 0.25× credits (not yet tested)

Configuration

  • Claude Code: CLAUDE_CODE_USE_BEDROCK=1 CLAUDE_CODE_DISABLE_THINKING=1 CLAUDE_CODE_MAX_TURNS=50
  • Claude Code flags: --bare --no-session-persistence --output-format json --permission-mode bypassPermissions
  • Kiro CLI: --no-interactive -a --effort high --model <model>
  • Kiro CLI Pro: $20/mo, 1000 credits included, overage $0.04/credit
